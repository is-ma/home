# Turismo de Software: 69 TUIs que Debes Probar Antes de Morir

Todo empezó con lazygit. Una tarde cualquiera, buscando una forma menos dolorosa de hacer rebase interactivo, descubrí que git podía sentirse como un videojuego. Paneles que se actualizan solos, navegación con teclas tipo vim, staging de líneas individuales sin memorizar flags oscuros. Fue la primera vez que pensé: "esto no es una herramienta, es un lugar al que quiero volver".

Esa sensación me llevó a buscar más. ¿Qué otras TUIs podrían ofrecer esa misma mezcla de utilidad y placer? Así nació el turismo de software: la idea de visitar interfaces de terminal como quien visita países. Cada TUI es un destino con su propia cultura, su propia lógica, su propia belleza. Algunas son como ciudades eficientes donde todo funciona a la perfección. Otras son como pueblos extraños con tradiciones curiosas. Todas merecen ser exploradas sin prisa, sin la presión de ser productivo, solo por el gusto de descubrir qué es posible cuando el texto es el único lienzo.

Esto no es una lista de herramientas para optimizar tu workflow. Es una invitación a perderse. Abre una terminal, instala cualquiera de estas TUIs y déjala correr sin un propósito concreto. Mira cómo btop dibuja gráficos de tu CPU. Navega por un cluster de Kubernetes con k9s. Lee un manga en manga-tui. Juega NetHack. Escucha música con cmus. Dibuja en PaTUI.

No son solo herramientas. Son experiencias. Y cada una merece ser vivida al menos una vez.

## Desarrollo

Esta sección reúne TUIs esenciales para el flujo de trabajo diario de desarrollo: desde gestión de Git y contenedores hasta edición de código y trabajo con APIs.

### lazygit
- **Descripción**: Interfaz terminal para git que simplifica operaciones complejas como rebase interactivo, cherry-pick y staging de líneas individuales.
- **Por qué gusta**: lazygit elimina la necesidad de memorizar decenas de flags oscuros de git. Su interfaz de paneles muestra el estado del repositorio de un vistazo: archivos modificados, ramas, historial de commits y cambios staged. El staging a nivel de hunk o línea individual es fluido con navegación tipo vim, y el rebase interactivo se convierte en una operación visual en lugar de un puzzle mental. La integración con GitHub muestra PRs directamente en la interfaz. Es la herramienta que hace que git deje de dar miedo.
- **Instalación**: `brew install lazygit`
- **Repo**: https://github.com/jesseduffield/lazygit

### lazydocker
- **Descripción**: Panel de control terminal para gestionar contenedores Docker, imágenes, volúmenes y servicios de docker-compose.
- **Por qué gusta**: lazydocker es como tener un htop dedicado exclusivamente a Docker. En lugar de encadenar `docker ps`, `docker logs`, `docker stats` y `docker compose up`, todo vive en una sola pantalla con navegación por teclado. Los logs se pueden hacer tail en tiempo real, las estadísticas de CPU/memoria de cada contenedor se muestran en gráficos, y reiniciar o eliminar contenedores es una pulsación de tecla.
- **Instalación**: `brew install lazydocker`
- **Repo**: https://github.com/jesseduffield/lazydocker

### k9s
- **Descripción**: Interfaz terminal para navegar, observar y gestionar clusters de Kubernetes en tiempo real.
- **Por qué gusta**: k9s transforma la experiencia de trabajar con kubectl de escribir comandos repetitivos a navegar un dashboard vivo. Los pods, servicios, deployments y nodos se actualizan en tiempo real. La función de "error zoom" lleva directamente al recurso con problemas, los logs se pueden tail con un enter, y el port-forwarding se activa sin salir de la interfaz.
- **Instalación**: `brew install derailed/k9s/k9s`
- **Repo**: https://github.com/derailed/k9s

### dive
- **Descripción**: Herramienta para explorar el contenido de cada capa de una imagen Docker y optimizar su tamaño.
- **Por qué gusta**: dive revela exactamente qué archivos se añaden, modifican o eliminan en cada capa de una imagen Docker. El panel izquierdo muestra las capas y el derecho el árbol de archivos con indicadores visuales de cambios. La métrica de "image efficiency" estima cuánto espacio se desperdicia por duplicación entre capas.
- **Instalación**: `brew install dive`
- **Repo**: https://github.com/wagoodman/dive

### yazi
- **Descripción**: Gestor de archivos ultrarrápido escrito en Rust con I/O asíncrono y preview de imágenes integrado.
- **Por qué gusta**: yazi es el file manager terminal más rápido que existe gracias a su arquitectura completamente asíncrona. El sistema de plugins en Lua permite personalizar prácticamente todo. La integración nativa con ripgrep, fd, fzf y zoxide lo convierte en un hub de productividad.
- **Instalación**: `brew install yazi`
- **Repo**: https://github.com/sxyazi/yazi

### broot
- **Descripción**: Navegador de árboles de directorios con búsqueda fuzzy que permite ver y manipular archivos manteniendo la jerarquía visible.
- **Por qué gusta**: broot resuelve el problema fundamental de navegar directorios grandes: nunca pierdes el contexto de la jerarquía mientras buscas. Los paneles múltiples permiten copiar/mover archivos entre directorios estilo Norton Commander. La búsqueda fuzzy filtra el árbol en tiempo real mostrando solo coincidencias pero manteniendo la estructura padre-hijo visible. Las acciones como renombrar, eliminar o crear archivos se ejecutan con atajos memorables y se pueden previsualizar antes de aplicar. La comunidad ha creado plugins para integrar broot con git, permitiendo ver cambios en el árbol mientras trabajas.
- **Instalación**: `brew install broot`
- **Repo**: https://github.com/Canop/broot

### helix
- **Descripción**: Editor modal de texto escrito en Rust con soporte LSP integrado y edición basada en selecciones múltiples al estilo Kakoune.
- **Por qué gusta**: helix elimina la necesidad de configurar plugins para tener un editor moderno funcional desde el primer momento. El soporte LSP viene integrado de serie: autocompletado, goto definition, rename y diagnostics funcionan sin instalar nada. El modelo de edición "selección primero, acción después" permite editar múltiples regiones del código simultáneamente con retroalimentación visual inmediata. Los temas y configuraciones se cargan desde archivos TOML simples, y la comunidad mantiene configs pre-hechas para docenas de lenguajes. La curva de aprendizaje es más suave que Vim porque las teclas son mnemotécnicas: `s` para seleccionar, `c` para cambiar, `d` para borrar.
- **Instalación**: `brew install helix`
- **Repo**: https://github.com/helix-editor/helix

### lnav
- **Descripción**: Visor de archivos de log que detecta formatos automáticamente, mergea múltiples archivos por timestamp y permite consultar con SQL.
- **Por qué gusta**: lnav resuelve el caos de analizar logs de múltiples servicios simultáneamente. Detecta automáticamente formatos, descomprime archivos al vuelo y mergea todo en una línea temporal unificada. La función más poderosa es poder hacer consultas SQL sobre los logs: `SELECT * FROM log WHERE level = 'ERROR'` filtra mensajes sin grep manual. El resaltado de sintaxis colorea diferentes niveles y servicios, facilitando escanear visualmente miles de líneas. Los usuarios avanzados crean filtros personalizados para enmascarar datos sensibles o resaltar patrones específicos de su aplicación.
- **Instalación**: `brew install lnav`
- **Repo**: https://github.com/tstack/lnav

### tmux
- **Descripción**: Multiplexor de terminal que permite crear, acceder y controlar múltiples sesiones de terminal desde una sola pantalla.
- **Por qué gusta**: tmux es la herramienta que hace posible trabajar remotamente sin miedo a perder el trabajo si se corta la conexión: las sesiones persisten en el servidor y se pueden reattach desde cualquier lugar. Los paneles permiten dividir la terminal en regiones independientes, cada una corriendo un proceso diferente. La funcionalidad de pair programming mediante sesiones compartidas permite colaborar en tiempo real con otros desarrolladores. Los plugins como tmux-resurrect guardan el estado completo de ventanas y paneles para restaurar tu workspace exactamente como lo dejaste. La comunidad mantiene cientos de plugins para notificaciones, integración con git, y temas personalizables.
- **Instalación**: `brew install tmux`
- **Repo**: https://github.com/tmux/tmux

### posting
- **Descripción**: Cliente HTTP moderno para terminal, alternativa a Postman/Insomnia con requests almacenados en archivos YAML.
- **Por qué gusta**: posting trae la comodidad de Postman al terminal sin sacrificar la eficiencia del teclado. Los requests se guardan en archivos YAML simples que se pueden versionar con git. La interfaz muestra headers, body y response en paneles separados con navegación fluida entre requests. Soporta autenticación OAuth2, variables de entorno y colecciones organizadas por proyecto. La integración con shell permite pipelear outputs directamente a otros comandos, algo imposible en clientes GUI.
- **Instalación**: `pipx install posting`
- **Repo**: https://github.com/darrenburns/posting

### gh-dash
- **Descripción**: Dashboard terminal para GitHub que muestra PRs e issues con secciones personalizables por repositorio.
- **Por qué gusta**: gh-dash convierte la gestión de PRs e issues de navegar la web de GitHub a un workflow de teclado fluido. Las acciones como ver diffs, hacer checkout, comentar, mergear y actualizar se ejecutan sin salir del terminal. Los filtros predefinidos muestran PRs que requieren tu review, los que tú abriste, o los que están bloqueados. La vista de diffs inline permite revisar cambios línea por línea con resaltado de sintaxis. La configuración permite definir secciones personalizadas por repositorio o equipo, ideal para mantener track de múltiples proyectos simultáneamente.
- **Instalación**: `gh extension install dlvhdr/gh-dash`
- **Repo**: https://github.com/dlvhdr/gh-dash

### jqp
- **Descripción**: Entorno interactivo TUI para experimentar con consultas jq, con preview de input y output en tiempo real.
- **Por qué gusta**: jqp elimina la frustración de construir queries jq complejos a ciegas. El panel de query permite escribir y ejecutar filtros con el resultado mostrado instantáneamente. El input se puede pegar directamente o cargar desde archivo, y el output se formatea con colores para identificar estructuras anidadas. La función de autocompletado sugiere filtros comunes como `map`, `select`, `reduce` mientras escribes. Los usuarios guardan queries frecuentes como snippets reutilizables, acelerando el trabajo con JSON en pipelines de datos.
- **Instalación**: `brew install jqp`
- **Repo**: https://github.com/noahgorstein/jqp

## Sistema y Monitoreo

Herramientas para monitoreo del sistema: recursos, GPU, red y disco en tiempo real.

### btop
- **Descripción**: Monitor de recursos con interfaz visual moderna que muestra CPU, memoria, discos, red y procesos en tiempo real.
- **Por qué gusta**: btop es el sucesor de bashtop y bpytop, reescrito en C++ para un rendimiento excelente. Ofrece gráficos de colores por núcleo de CPU y gestión de procesos integrada. Los temas personalizables permiten cambiar completamente la apariencia con esquemas de colores cyberpunk, minimalistas o retro. Las alertas configurables notifican cuando CPU, memoria o disco superan umbrales definidos. La función de historial guarda métricas para analizar tendencias, y los usuarios pueden exportar datos a formatos legibles para generar reportes.
- **Instalación**: `sudo apt install btop`
- **Repo**: https://github.com/aristocratos/btop

### htop
- **Descripción**: Visor interactivo de procesos multiplataforma, estándar de facto para inspección y gestión de procesos en terminal.
- **Por qué gusta**: htop lleva más de 20 años siendo la herramienta de referencia. Permite scroll vertical y horizontal, vista en árbol, búsqueda incremental y matar procesos sin escribir PIDs. La vista de árbol muestra relaciones padre-hijo entre procesos, revelando qué aplicación spawnó cada hijo. Los contadores personalizados permiten monitorear métricas específicas como threads abiertos o uso de swap. La comunidad ha creado forks como btop pero htop sigue siendo el estándar por su estabilidad y presencia en todos los repositorios oficiales.
- **Instalación**: `sudo apt install htop`
- **Repo**: https://github.com/htop-dev/htop

### glances
- **Descripción**: Herramienta de monitoreo cross-platform que presenta CPU, memoria, disco, red, contenedores, sensores y más en un dashboard unificado.
- **Por qué gusta**: Glances destaca por su versatilidad: funciona como TUI local, servidor web remoto, o cliente/servidor con API RESTful. El modo servidor web permite monitorear máquinas remotas desde cualquier navegador sin instalar clientes adicionales. La integración con Docker muestra estadísticas de contenedores en tiempo real junto a métricas del host. Los exports a CSV, JSON o InfluxDB facilitan la integración con sistemas de monitoreo enterprise. Los plugins extienden funcionalidades para monitorear servicios específicos como Nginx, MySQL o RabbitMQ.
- **Instalación**: `pip install 'glances[all]'`
- **Repo**: https://github.com/nicolargo/glances

### nvtop
- **Descripción**: Monitor de GPUs y aceleradores estilo htop para NVIDIA, AMD, Intel, Apple M1/M2, Qualcomm Adreno y más.
- **Por qué gusta**: nvtop llena un hueco que ningún otro TUI cubre: monitoreo dedicado de GPUs con gráficos de uso, memoria, temperatura y procesos por GPU. Soporta múltiples GPUs simultáneamente, mostrando cada una en paneles separados con métricas individuales. La vista de procesos muestra qué aplicaciones están usando la GPU y cuánto memoria VRAM consumen. Los gráficos históricos revelan picos de uso durante rendering, entrenamiento de ML o gaming. El soporte para GPUs integradas de Intel y Apple Silicon lo hace útil incluso en laptops sin GPU dedicada.
- **Instalación**: `sudo apt install nvtop`
- **Repo**: https://github.com/Syllo/nvtop

### bandwhich
- **Descripción**: Utilidad que muestra el consumo de ancho de banda de red desglosado por proceso, conexión e IP remota.
- **Por qué gusta**: bandwhich responde a la pregunta "¿qué proceso está consumiendo mi ancho de banda?" en tiempo real. Sniffea la interfaz de red y cruza el tamaño de paquetes con /proc. La vista por proceso agrupa todo el tráfico de cada aplicación, revelando qué programa está descargando en segundo plano. La resolución DNS integrada muestra nombres de dominio en lugar de solo IPs, facilitando identificar conexiones sospechosas. El filtro por interfaz permite monitorear solo WiFi, Ethernet o interfaces específicas en máquinas con múltiples conexiones.
- **Instalación**: `cargo install bandwhich --locked`
- **Repo**: https://github.com/imsnif/bandwhich

### trippy
- **Descripción**: Herramienta de diagnóstico de red que combina traceroute y ping en una interfaz TUI interactiva.
- **Por qué gusta**: trippy fusiona lo mejor de traceroute y mtr en una sola herramienta con visualización gráfica de cada salto. Muestra latencia, pérdida de paquetes y estadísticas por hop. El modo continuo envía paquetes constantemente, actualizando las métricas en tiempo real para detectar problemas intermitentes. Los gráficos de latencia por salto revelan cuellos de botella específicos en la ruta de red. La exportación a CSV permite analizar datos históricos o integrar con dashboards de monitoreo externos.
- **Instalación**: `cargo install trippy --locked`
- **Repo**: https://github.com/fujiapple852/trippy

### gping
- **Descripción**: Ping visual con gráfico en tiempo real que muestra la latencia como una curva animada en terminal.
- **Por qué gusta**: gping transforma el ping aburrido en un gráfico visual inmediato donde se ven picos de latencia de un vistazo. Permite monitorear múltiples hosts simultáneamente. El gráfico se actualiza en tiempo real con cada respuesta, mostrando tendencias de latencia que números crudos ocultarían. Los usuarios pueden comparar rutas de red lado a lado, útil para diagnosticar problemas de conexión intermitentes. La salida es perfecta para demos en vivo o para dejar corriendo en un panel mientras se trabaja en otras tareas.
- **Instalación**: `brew install gping`
- **Repo**: https://github.com/orf/gping

### ncdu
- **Descripción**: Analizador de uso de disco basado en ncurses que muestra qué directorios consumen más espacio de forma rápida e interactiva.
- **Por qué gusta**: ncdu es el estándar para encontrar "space hogs" en servidores remotos. Escanea directorios y permite navegar interactivamente por la jerarquía, ordenar por tamaño, y eliminar archivos. La navegación con teclado tipo vim permite moverse rápidamente entre directorios sin tocar el mouse. El modo read-only previene eliminaciones accidentales en sistemas críticos. Los reportes exportables muestran qué directorios han crecido más entre escaneos, útil para tracking de uso a lo largo del tiempo.
- **Instalación**: `sudo apt install ncdu`
- **Repo**: https://dev.yorhel.nl/ncdu

### gdu
- **Descripción**: Analizador de uso de disco ultrarrápido escrito en Go, optimizado para SSDs con procesamiento paralelo.
- **Por qué gusta**: gdu destaca por su velocidad brutal gracias al escaneo paralelo nativo de Go. Ofrece interfaz TUI interactiva con navegación por directorios. El escaneo paralelo utiliza múltiples cores para leer el sistema de archivos simultáneamente, reduciendo tiempos de espera en discos grandes. La integración con stdin permite pipelear listas de archivos desde otros comandos para análisis personalizados. Los atajos de teclado permiten eliminar archivos directamente desde la interfaz, aunque el modo solo-lectura está disponible para entornos productivos.
- **Instalación**: `sudo apt install gdu`
- **Repo**: https://github.com/dundee/gdu

### duf
- **Descripción**: Alternativa moderna y colorida al comando `df` que muestra uso y espacio libre de sistemas de archivos de forma clara.
- **Por qué gusta**: duf transforma la salida cruda de `df` en tablas coloridas y organizadas. Agrupa dispositivos automáticamente con filtros para mostrar u ocultar categorías. Los indicadores visuales de colores muestran de un vistazo qué filesystems están cerca de llenarse. El formato de tabla alinea columnas perfectamente incluso con nombres de dispositivos largos. Las opciones de filtrado permiten mostrar solo filesystems locales, remotos, o de tipos específicos como tmpfs o overlay.
- **Instalación**: `sudo apt install duf`
- **Repo**: https://github.com/muesli/duf

## Productividad

Herramientas TUI para aumentar la productividad en gestión de datos, tareas y presentaciones.

### Datos

#### VisiData
- **Descripción**: Hoja de cálculo en terminal para explorar, filtrar y transformar datos tabulares en formatos CSV, JSON, Excel, SQLite y más.
- **Por qué gusta**: Permite hacer tablas de frecuencia, pivotar columnas, crear histogramas y aplicar fórmulas directamente sobre los datos sin salir de la terminal. La navegación con atajos de teclado permite moverse por datasets de millones de filas sin lag. Los filtros se aplican en tiempo real con búsqueda fuzzy, y las columnas se pueden ocultar, renombrar o reordenar al vuelo. La función de join permite combinar múltiples archivos como en SQL, y los resultados se pueden exportar a cualquier formato soportado. La comunidad mantiene un repositorio de plugins para visualizar mapas, conectar a APIs REST, y generar reportes.
- **Instalación**: `pipx install visidata`
- **Repo**: https://github.com/saulpw/visidata

### Tareas

#### taskwarrior-tui
- **Descripción**: Interfaz terminal para Taskwarrior que permite gestionar tareas con atajos de teclado y vista visual de proyectos.
- **Por qué gusta**: Combina la potencia de Taskwarrior como motor de tareas con una interfaz interactiva que muestra tareas organizadas por proyecto, contexto y prioridad. El panel lateral muestra estadísticas de tareas completadas vs pendientes por proyecto. La creación rápida de tareas con autocompletado de tags y due dates acelera el capture de ideas. Los filtros predefinidos permiten ver tareas urgentes, vencidas, o bloqueadas por dependencias. La integración con calendarios muestra deadlines en una vista mensual, ayudando a planificar sprints de trabajo.
- **Instalación**: `brew install taskwarrior-tui`
- **Repo**: https://github.com/kdheepak/taskwarrior-tui

### Presentaciones

#### slides
- **Descripción**: Herramienta de presentaciones en terminal que convierte archivos Markdown en diapositivas navegables.
- **Por qué gusta**: Cada separador `---` en un archivo Markdown se convierte automáticamente en una nueva diapositiva. Soporte nativo para bloques de código con resaltado de sintaxis. La navegación con flechas o teclas vim permite avanzar/retroceder suavemente durante presentaciones. El modo presentador muestra notas ocultas en una segunda terminal o ventana. Los temas personalizables cambian colores y bordes para adaptar el look a diferentes audiencias o marcas corporativas.
- **Instalación**: `brew install slides`
- **Repo**: https://github.com/maaslalani/slides

#### gum
- **Descripción**: Colección de utilidades TUI para crear scripts de shell interactivos con inputs, selecciones, confirmaciones y tablas.
- **Por qué gusta**: Transforma scripts bash aburridos en experiencias interactivas con `gum choose`, `gum input`, `gum confirm` y `gum filter`. Cada utilidad produce output formateado listo para capturar en variables de shell. El spinner y progress bar dan feedback visual durante operaciones largas, mejorando la UX de scripts. Los estilos personalizables con colores y bordes permiten crear interfaces consistentes con tu branding. La comunidad comparte snippets para menús de despliegue, wizards de configuración y prompts de CI/CD.
- **Instalación**: `brew install gum`
- **Repo**: https://github.com/charmbracelet/gum

### Historial

#### atuin
- **Descripción**: Reemplazo del historial del shell con base de datos SQLite, búsqueda avanzada y sincronización cifrada entre máquinas.
- **Por qué gusta**: Reemplaza Ctrl+R con una interfaz de pantalla completa que busca en todo el historial con filtros por directorio, sesión, fecha y código de salida. La sincronización cifrada permite acceder al mismo historial desde todas tus máquinas sin exponer comandos sensibles. Las estadísticas muestran comandos más usados, horas productivas y patrones de trabajo a lo largo del tiempo. La función de "session recording" permite replay de sesiones completas para debugging o aprendizaje. Los plugins integran atuin con fuzzy finders externos para búsquedas aún más potentes.
- **Instalación**: `curl --proto '=https' --tlsv1.2 -LsSf https://setup.atuin.sh | sh`
- **Repo**: https://github.com/atuinsh/atuin

## Multimedia

TUIs para música y lectura en terminal.

### Música

#### cmus
- **Descripción**: Reproductor de música ligero y ultrarrápido para sistemas Unix, escrito en C.
- **Por qué gusta**: cmus lleva más de 20 años siendo el reproductor de referencia en terminal. Su consumo de memoria es ridículo, arranca al instante y reproduce cualquier formato. La biblioteca organiza automáticamente por artista, álbum, género y año con navegación fluida entre vistas. Los atajos de teclado permiten crear playlists dinámicas, ajustar ecualización y controlar volumen sin salir de la interfaz. El soporte para temas y scripts permite personalizar la apariencia y automatizar tareas como descargar letras o metadata.
- **Instalación**: `sudo apt install cmus`
- **Repo**: https://github.com/cmus/cmus

#### ytui-music
- **Descripción**: Cliente de YouTube Music en terminal con interfaz TUI elegante, escrito en Rust.
- **Por qué gusta**: ytui-music permite buscar y reproducir música de YouTube directamente desde la terminal sin necesidad de navegador. Usa mpv como motor de reproducción. La búsqueda integrada accede al catálogo completo de YouTube Music con sugerencias mientras escribes. Las playlists se pueden crear, editar y sincronizar con tu cuenta de Google. El modo sin anuncios y la reproducción en segundo plano lo hacen ideal para sesiones de coding largas sin distracciones visuales.
- **Instalación**: `cargo install ytui-music`
- **Repo**: https://github.com/sudipghimire533/ytui-music

### Lectura

#### manga-tui
- **Descripción**: Lector y descargador de manga en terminal con renderizado de imágenes, escrito en Rust.
- **Por qué gusta**: manga-tui es probablemente el lector de manga en terminal más completo. Soporta múltiples proveedores y las imágenes se renderizan directamente en terminales compatibles. La búsqueda por título, autor o género permite descubrir nuevas series sin salir de la terminal. El modo descarga guarda capítulos completos para lectura offline, ideal para viajes sin conexión. El historial de lectura marca automáticamente el último capítulo leído por serie.
- **Instalación**: `cargo install manga-tui --locked`
- **Repo**: https://github.com/josueBarretogit/manga-tui

#### bookokrat
- **Descripción**: Lector de libros electrónicos en terminal que soporta EPUB, PDF y DJVU, escrito en Rust.
- **Por qué gusta**: bookokrat destaca por combinar tres formatos de lectura en una sola aplicación. Soporta marcadores, búsqueda dentro del texto y estadísticas de lectura. ## Comunicación y Social

Clientes de chat, redes sociales y foros que viven en la terminal. IRC, Matrix, Mastodon, Reddit y Discord sin distracciones.

### WeeChat
- **Descripción**: Cliente IRC extensible y modular, el estándar de facto en terminales desde 2003.
- **Por qué gusta**: WeeChat es el veterano indiscutible del IRC con más de 20 años de desarrollo activo. Su arquitectura de plugins permite cargar scripts en 7 lenguajes sin tocar el núcleo. El sistema de buffers organiza canales y privados en pestañas navegables con atajos de teclado. Las notificaciones integradas avisan de menciones y mensajes privados incluso en segundo plano. La comunidad mantiene cientos de scripts para integración con GitHub, Twitter, notificaciones de sistema y más.
- **Instalación**: `sudo apt install weechat`
- **Repo**: https://github.com/weechat/weechat

### irssi
- **Descripción**: Cliente IRC modular en modo texto, creado en 1999, con sistema de temas y scripting en Perl.
- **Por qué gusta**: irssi es el abuelo de los TUIs de chat y sigue vivo gracias a su simplicidad brutal. Su sistema de ventanas permite dividir la terminal en múltiples paneles independientes. Los temas personalizables cambian colores y formatos de mensajes para adaptar la interfaz a preferencias visuales. El scripting en Perl permite automatizar respuestas, logging personalizado e integración con servicios externos. Los usuarios avanzados configuran irssi para correr en screen/tmux en servidores remotos, accediendo via SSH desde cualquier lugar.

- **Repo**: https://github.com/irssi/irssi

### Repartee
- **Descripción**: Cliente IRC moderno escrito en Rust con Ratatui, cifrado E2E nativo y frontend web opcional.
- **Por qué gusta**: Repartee es la opción moderna para quien quiere IRC sin sacrificar seguridad. Su protocolo RPE2E ofrece cifrado de extremo a extremo para mensajes privados y canales. La interfaz Ratatui es moderna y responsiva, con soporte para temas y layouts personalizables. El frontend web opcional permite acceder al mismo backend desde navegador cuando no estás en tu terminal principal. La arquitectura en Rust garantiza rendimiento y seguridad de memoria, eliminando vulnerabilidades comunes en clientes heredados.
- **Instalación**: `cargo install repartee`

### gomuks
- **Descripción**: Cliente Matrix escrito en Go con cifrado E2E, backend separado y frontend terminal experimental.
- **Por qué gusta**: gomuks es el cliente Matrix TUI más maduro disponible. Su arquitectura de backend separado actúa como un bouncer: puedes conectarte desde la TUI, web o Android al mismo backend. El cifrado E2E protege mensajes privados y canales cifrados contra interceptación. La sincronización entre dispositivos es instantánea, mostrando el mismo estado en todas las sesiones. Las notificaciones push avisan de mensajes nuevos incluso con la TUI cerrada, ideal para stay connected sin mantener sesiones abiertas.

### Mastui

El renderizado de imágenes inline muestra fotos y memes directamente en el terminal compatible. La composición de toots soporta hilos, encuestas y contenido trigger warnings. Los atajos de teclado son intuitivos y personalizables, con Vim mode opcional para usuarios avanzados.

### tut

Los atajos de teclado son eficientes una vez memorizados: `o` para abrir, `r` para reply, `b` para boost. El soporte para múltiples cuentas permite switchear entre perfiles personales y profesionales. ## Creativas y Diversión

Aquí no se trata de productividad: se trata de hacer música, dibujar píxeles, crear animaciones ASCII y parecer un hacker de película.

### Imbolc Studio
- **Descripción**: Estación de trabajo de audio digital (DAW) completa que corre enteramente en la terminal, con 58 instrumentos y 39 efectos integrados.
- **Por qué gusta**: Imbolc demuestra que la terminal puede ser un estudio de música serio. Se conecta a SuperCollider y ofrece un mixer con hasta 28 buses, edición de patrones y síntesis en tiempo real. Los instrumentos incluyen sintetizadores, samplers y drum machines programables. Los efectos van desde EQ y compresión hasta reverberación y delay, todos configurables via TUI. El secuenciador permite crear canciones completas con múltiples pistas y exportar a formatos de audio estándar.
- **Instalación**: `cargo install imbolc-studio`
- **Repo**: https://github.com/mohsenil85/imbolc

### textStep
- **Descripción**: Secuenciador por pasos, caja de ritmos y sintetizador polifónico dual construido completamente en Rust con DSP escrito desde cero.
- **Por qué gusta**: textStep no usa samples ni librerías de audio externas. Ofrece 8 pistas de drums y dos sintetizadores polifónicos independientes con crossfader estilo DJ. Cada sintetizador tiene osciladores, filtros y envolventes configurables en tiempo real. Los patrones se programan paso a paso con resolución ajustable, y las canciones pueden combinar múltiples patrones en secuencia. El motor de audio de baja latencia permite tocar en vivo sin delays perceptibles.
- **Instalación**: `cargo install textstep`
- **Repo**: https://github.com/illobo/textStep

### PaTUI
- **Descripción**: Editor de imágenes en terminal inspirado en MS Paint con controles de navegación estilo Vim.
- **Por qué gusta**: PaTUI carga imágenes PNG o JPEG y las renderiza como caracteres de bloque coloreados. Los controles son Vim puro: `i` para modo pintura, `hjkl` para mover, `dd` para borrar. Las herramientas incluyen pincel, línea, rectángulo, círculo y relleno. Los colores se seleccionan de una paleta de 256 colores ANSI. Las creaciones se pueden exportar como PNG manteniendo la fidelidad visual del arte terminal.
- **Instalación**: `brew install georgemandis/tap/patui`
- **Repo**: https://github.com/georgemandis/patui

### Durdraw
- **Descripción**: Editor de arte ASCII, ANSI y Unicode con animación frame a frame, 256 colores, soporte de ratón y temas personalizables.
- **Por qué gusta**: Durdraw es el editor de arte ANSI más completo disponible en terminal. Inspirado en clásicos como TheDraw pero con capacidades modernas: animación, 256 colores y exportación a múltiples formatos. El sistema de animación permite crear GIFs frame a frame con preview en tiempo real. Las herramientas de dibujo incluyen formas, texto, spray y selección. Los proyectos se guardan en formato abierto y se pueden exportar a PNG, GIF o videos para compartir fuera del terminal.
- **Instalación**: `pip3 install durdraw`
- **Repo**: https://github.com/cmang/durdraw

### aaa (3a)
- **Descripción**: Navaja suiza para arte ASCII animado: lee, escribe, edita, genera, reproduce y convierte animaciones 3a a SVG, PNG, GIF, WebP, MP4 y más.
- **Por qué gusta**: aaa trabaja con el formato 3a (Animated ASCII Art), un formato de texto plano que preserva el formato visual en cualquier editor. Ofrece conversión a múltiples formatos. El editor integrado permite crear y modificar animaciones frame a frame con herramientas de dibujo ASCII. La función de generate crea animaciones procedurales a partir de descripciones textuales. Los artistas comparten sus creaciones en GitHub, y aaa permite importar/exportar fácilmente para remixar trabajos de otros.
- **Instalación**: `cargo install aaa`
- **Repo**: https://github.com/asciimoth/aaa

### cmatrix
- **Descripción**: Simula la icónica lluvia de caracteres verdes de la película "The Matrix" directamente en la terminal.
- **Por qué gusta**: cmatrix es probablemente el TUI más reconocible de la historia de Linux. Genera columnas de caracteres aleatorios que caen a velocidades variables sobre fondo negro. Los modos personalizables permiten cambiar colores, densidad decolumnas y caracteres usados (katakana, latino, binario). El efecto de parpadeo y desvanecimiento crea una animación hipnótica perfecta para demos o screensavers. Es el comando clásico para impresionar a no-técnicos o llenar pantallas en convenciones.

- **Repo**: https://github.com/abishekvashok/cmatrix

### Asciiquarium

## Navegadores y Ciencia

Navegadores que funcionan sobre SSH y herramientas científicas que demuestran que la terminal puede ser más que texto.

### Carbonyl
- **Descripción**: Navegador basado en Chromium que renderiza páginas web completas directamente en la terminal, con soporte para WebGL, vídeo y animaciones a 60 FPS.
- **Por qué gusta**: Carbonyl no es un navegador de texto convencional: es Chromium real adaptado para funcionar sin servidor gráfico. Arranca en menos de un segundo y reproduce vídeos de YouTube. El renderizado usa protocolos de terminal modernos (Kitty, Sixel) para mostrar imágenes y gráficos complejos. El soporte para JavaScript permite ejecutar aplicaciones web completas como Twitter, GitHub o Google Docs. Es ideal para navegar sobre SSH o en entornos sin X11/Wayland disponible.
- **Instalación**: `npm install --global carbonyl`
- **Repo**: https://github.com/fathyb/carbonyl

### Browsh
- **Descripción**: Navegador de texto moderno que usa Firefox real como motor de renderizado, compatible con HTML5, CSS3, JavaScript y WebGL.
- **Por qué gusta**: Browsh resuelve el problema que ningún navegador de texto tradicional puede: ejecutar JavaScript real. Su caso de uso estrella es navegar por SSH en conexiones de bajo ancho de banda. El renderizado convierte páginas completas a caracteres ANSI preservando layout y colores. El modo texto elimina imágenes y vídeos para ahorrar ancho de banda, mostrando solo contenido esencial. La integración con Firefox permite usar extensiones y sincronizar bookmarks con tu perfil existente.
- **Instalación**: `brew install browsh`
- **Repo**: https://github.com/browsh-org/browsh

### Chawan
- **Descripción**: Sucesor moderno de w3m escrito en Nim, con soporte para CSS avanzado, imágenes inline y JavaScript mediante QuickJS.
- **Por qué gusta**: Chawan implementa su propio motor de renderizado desde cero con soporte para flexbox, floats y tablas CSS. Las imágenes inline funcionan mediante protocolos Sixel o Kitty. El soporte para JavaScript via QuickJS permite ejecutar scripts básicos y aplicaciones web modernas. El modo línea permite integrar Chawan en pipelines Unix, extrayendo texto formateado de páginas web. Es el navegador de terminal más cercano a una experiencia web completa sin dependencias de GUI.
- **Instalación**: `brew install chawan`
- **Repo**: https://codeberg.org/bptato/chawan

### ncalc
- **Descripción**: Calculadora científica y sistema de álgebra computacional (CAS) para terminal, con cálculo simbólico y gráficos en patrones Braille Unicode.
- **Por qué gusta**: ncalc combina el poder de GiNaC con una interfaz ncurses intuitiva. Permite resolver ecuaciones algebraicas, calcular derivadas simbólicas y graficar con zoom interactivo. Los gráficos en Braille Unicode muestran funciones 2D y 3D con resolución sorprendentemente alta. El historial de cálculos permite revisar y reutilizar expresiones anteriores. El soporte para variables y funciones definidas por usuario facilita cálculos repetitivos en física o ingeniería.
- **Instalación**: `sudo apt install ncalc`
- **Repo**: https://github.com/jt-copernicus/ncalc

### gnuplot
- **Descripción**: Utilidad de graficado científico interactiva por línea de comandos, en desarrollo activo desde 1986, con soporte para gráficos 2D y 3D.
- **Por qué gusta**: gnuplot es el estándar de facto para visualización científica en terminal desde hace casi 40 años. Su modo terminal `set term dumb` renderiza gráficos ASCII directamente. Los scripts automatizan generación de gráficos para papers o reportes técnicos. El soporte para ajuste de curvas y regresiones permite analizar datos experimentales sin software propietario. Las salidas se pueden exportar a formatos vectoriales (SVG, PDF) o raster (PNG) para publicación.
- **Instalación**: `sudo apt install gnuplot`
- **Repo**: https://github.com/gnuplot/gnuplot

- **Descripción**: Multiplexor de terminal moderno en Rust con plugins WASM, paneles flotantes y sesiones colaborativas para pair programming.
- **Por qué está aquí**: No es un multiplexor al uso: su sistema de plugins WASM permite extenderlo como un entorno de desarrollo completo, pero también es una herramienta de productividad con layouts configurables y sesiones persistentes, y al mismo tiempo una experiencia de usuario que compite con tmux en funcionalidad con una curva de aprendizaje mucho más suave. Los plugins se escriben en cualquier lenguaje que compile a WebAssembly, desde Rust hasta Go y AssemblyScript. El layout system guarda configuraciones de paneles en archivos YAML que se versionan con git, permitiendo compartir entornos de trabajo entre equipos. Las sesiones compartidas habilitan pair programming sin configurar servidores adicionales ni herramientas externas. Es demasiado transversal para una sola categoría.
- **Instalación**: `brew install zellij`
- **Repo**: https://github.com/zellij-org/zellij

### cadubi
- **Descripción**: Utilidad de dibujo ASCII con colores, negrita, inverso y parpadeo.
- **Por qué está aquí**: Está en Creativas pero es también un editor de texto especializado: dibujas píxeles con caracteres pero la navegación y atributos funcionan como un editor tradicional.
- **Instalación**: `sudo apt install cadubi`
- **Repo**: https://github.com/statico/cadubi

### ncmpcpp
- **Descripción**: Cliente ncurses con interfaz rica para el servidor de música MPD.
- **Por qué está aquí**: No es un reproductor standalone: requiere MPD corriendo como demonio separado, lo que lo convierte más en un panel de control remoto que en una aplicación de música independiente.
- **Instalación**: `sudo apt install ncmpcpp`
- **Repo**: https://github.com/ncmpcpp/ncmpcpp

### hollywood
- **Descripción**: Divide la terminal en múltiples paneles con tecno-babble para simular pantallas de hacker.
- **Por qué está aquí**: Es puramente experiencia estética: no produces nada, no resuelves problemas, solo transformas tu terminal en el centro de mando de una película de espías.
- **Instalación**: `sudo apt install hollywood`
- **Repo**: https://github.com/dustinkirkland/hollywood

---

## Conclusión

Estas diez TUIs demuestran que el software de terminal no siempre encaja en cajas limpias. Algunas son navajas suizas transversales, otras son experiencias más que herramientas, y algunas simplemente viven en los bordes entre categorías. Esto es solo el principio: existen cientos de TUIs por descubrir, desde clientes de email hasta simuladores de vuelo, desde editores de vídeo hasta laboratorios de química computacional. La terminal es más vasta de lo que parece. Si encuentras una TUI que te sorprenda, compártela: el turismo de terminal es un territorio que se mapea entre todos.
